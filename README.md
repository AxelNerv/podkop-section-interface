# Podkop — патч «Интерфейс секции»

Добавляет в LuCI Podkop новое поле **«Интерфейс секции»** прямо в форму каждой секции.  
При выборе интерфейса его подсеть автоматически добавляется в маршрутизацию — весь трафик с этой подсети идёт через данную секцию без ручного ввода IP.

**Поддерживается:** Podkop v0.7.17+, OpenWrt 23.x / 24.x

---

## Что делает патч

Без патча единственный способ направить весь трафик конкретной подсети через секцию — вручную вписывать IP/подсеть в поле «Полностью маршрутизированные IP-адреса». Патч автоматизирует это через выбор имени интерфейса.

**После установки патча:**
- В каждой секции типа **Proxy** или **VPN** появляется поле **«Интерфейс секции»** и выпадающий список **«Режим интерфейса секции»**
- Доступные значения интерфейса: `br-lan`, `vlan2`, `vlan3` или любое своё
- При старте Podkop читает UCI опции `section_interface` / `section_interface_mode`, автоматически определяет реальное имя устройства через `ifstatus` и настраивает маршрутизацию

**Два режима работы:**

| Режим | Поведение |
|-------|-----------|
| **Только списки секции** *(по умолчанию)* | Через секцию идёт только трафик к доменам/подсетям из списков этой секции. Пример: в секции выбран список YouTube → с этой точки доступа YouTube идёт через VPN, всё остальное — напрямую. |
| **Весь трафик подсети** | Весь трафик с интерфейса идёт через секцию. Работает даже если списки секции пустые — удобно для «полностью VPN» точки доступа. |

**Пример лога (режим «Только списки секции»):**
```
podkop: [info] section main: lists scoped to source 192.168.4.1/24 (interface vlan2)
```

**Пример лога (режим «Весь трафик подсети»):**
```
podkop: [info] section main: adding subnet 192.168.4.1/24 from interface vlan2 to fully routed
```

**Если поле оставить пустым** — ничего не меняется, секция работает как обычно.

### Сценарий: две Wi-Fi точки — обычная и VPN

1. **Обычная точка (без VPN):** привяжи её к сети, чьё устройство **не добавлено** в `source_network_interfaces` Podkop — трафик идёт напрямую, Podkop его не трогает.
2. **VPN-точка:** создай vlan2 по гайду ниже, добавь её устройство в `source_network_interfaces`, укажи `vlan2` в поле «Интерфейс секции» нужной секции и выбери режим:
   - «Весь трафик подсети» — всё через VPN (списки можно не заполнять)
   - «Только списки секции» — через VPN только сервисы из списков

**Какой интерфейс выбрать:**
| Интерфейс | Что это |
|-----------|---------|
| `br-lan` | Основная LAN — весь кабельный трафик (ПК, телевизоры и т.д.) |
| `vlan2` | Отдельный VLAN, например для гостевой сети или второго сегмента |
| `vlan3` | Ещё один VLAN |
| любое имя | Любой интерфейс роутера — можно ввести вручную |

## Бэкап

Скрипт `install.sh` делает бэкап автоматически в `/etc/podkop-patch-backup/`.

**Ручной бэкап** (если хочешь подстраховаться перед запуском):

```sh
ssh root@192.168.1.1

mkdir -p /etc/podkop-patch-backup
cp /usr/bin/podkop                                    /etc/podkop-patch-backup/podkop.orig
cp /www/luci-static/resources/view/podkop/section.js  /etc/podkop-patch-backup/section.js.orig

ls -la /etc/podkop-patch-backup/
```

**Восстановление вручную** (если что-то пошло не так):

```sh
cp /etc/podkop-patch-backup/podkop.orig     /usr/bin/podkop
cp /etc/podkop-patch-backup/section.js.orig /www/luci-static/resources/view/podkop/section.js
chmod +x /usr/bin/podkop
/etc/init.d/podkop restart
```

## Установка

Подключиться по SSH к роутеру и выполнить одну команду:

```sh
wget -q https://raw.githubusercontent.com/AxelNerv/podkop-section-interface/main/install.sh -O /tmp/install.sh && sh /tmp/install.sh
```

Скрипт автоматически:
1. Проверит что Podkop установлен
2. Создаст бэкапы оригинальных файлов
3. Скачает и применит патч
4. Проверит синтаксис bash перед заменой

После установки нажать **Ctrl+Shift+R** в браузере (сброс кэша LuCI).

---

## Использование

1. Открыть LuCI: `http://192.168.1.1` → **Службы → Podkop**
2. Вкладка **Секции**
3. Выбрать нужную секцию
4. Найти поле **«Интерфейс секции»**
5. Выбрать интерфейс из списка или ввести своё название
6. **Сохранить и применить**
7. Перезапустить Podkop: `/etc/init.d/podkop restart`

> Поле отображается только для секций с типом подключения **Proxy** или **VPN**.

## Удаление

```sh
wget -q https://raw.githubusercontent.com/AxelNerv/podkop-section-interface/main/uninstall.sh -O /tmp/uninstall.sh && sh /tmp/uninstall.sh
```

После удаления: **Ctrl+Shift+R** в браузере, рестарт Podkop.

## Создание отдельной Wi-Fi сети (vlan2)

Если у тебя одна Wi-Fi сеть и ты хочешь добавить вторую (например, чтобы назначить её секции Podkop) — вот пошаговая инструкция через LuCI.

### Шаг 1 — Создать сетевой интерфейс

**LuCI → Сеть → Интерфейсы → Добавить новый интерфейс**

| Поле | Значение |
|------|---------|
| Имя интерфейса | `vlan2` |
| Протокол | Статический адрес |
| Устройство | *(оставь пустым / «не указано» — привяжется после создания Wi-Fi)* |

Перейди на вкладку **«Общие настройки»**:
- IPv4-адрес: `192.168.4.1`
- Маска подсети: `255.255.255.0`

Перейди на вкладку **«DHCP-сервер»** → включи:
- Диапазон адресов: начало `100`, количество `150`

### Шаг 1.5 — Зона файрвола (ОБЯЗАТЕЛЬНО!)

> ⚠️ **Самая частая ошибка.** Без этого шага телефон будет видеть сеть, но не сможет подключиться («Не удалось подключиться» / «Ошибка получения IP-адреса»). Причина: в OpenWrt входящий трафик с интерфейса без зоны режется политикой по умолчанию (`input REJECT`) — DHCP-запросы до роутера просто не доходят.

В том же окне интерфейса перейди на вкладку **«Настройки межсетевого экрана»**:
- **Создать / назначить зону межсетевого экрана** → выбери `lan`

**Сохранить и применить.**

> Если хочешь изолировать новую сеть от основной — вместо `lan` создай отдельную зону (впиши имя `vlan2` в это же поле), а затем в **Сеть → Межсетевой экран** для зоны `vlan2` поставь: «Входящий трафик» = Принимать, «Перенаправление» = Отклонять, и добавь разрешение перенаправления `vlan2 → wan`.

### Шаг 2 — Создать Wi-Fi точку доступа

**LuCI → Сеть → Беспроводные сети** → найди нужное радио (обычно `radio0` или `radio1`) → кнопка **«Добавить»**

| Поле | Значение |
|------|---------|
| Имя SSID | `OpenWrt1` *(или любое)* |
| Режим | Точка доступа (AP) |
| Сеть | `vlan2` *(выбери интерфейс, созданный в шаге 1)* |
| Безопасность | WPA2-PSK, задай пароль |

**Сохранить и применить.**

### Шаг 3 — Узнать реальное имя устройства

После применения настроек выполни:
```sh
ifstatus vlan2 | jsonfilter -e '@.l3_device'
# → phy1-ap1  (или другое имя)
```

### Шаг 4 — Добавить в Podkop

```sh
# Замени phy1-ap1 на имя из предыдущего шага
uci add_list podkop.settings.source_network_interfaces=phy1-ap1
uci commit podkop
/etc/init.d/podkop restart
```

### Шаг 5 — Назначить секцию

1. LuCI → Службы → Podkop → вкладка **Секции**
2. Открой нужную секцию
3. В поле **«Интерфейс секции»** — выбери `vlan2` из выпадающего списка
4. Сохранить и применить → рестарт Podkop

## Альтернатива: настройка одной командой через SSH

Если не хочется кликать по LuCI — шаги 1 и 1.5 можно сделать одним блоком. Подключись по SSH и выполни:

```sh
# Интерфейс vlan2
uci set network.vlan2=interface
uci set network.vlan2.proto='static'
uci set network.vlan2.ipaddr='192.168.4.1'
uci set network.vlan2.netmask='255.255.255.0'

# DHCP для vlan2
uci set dhcp.vlan2=dhcp
uci set dhcp.vlan2.interface='vlan2'
uci set dhcp.vlan2.start='100'
uci set dhcp.vlan2.limit='150'
uci set dhcp.vlan2.leasetime='12h'

# Добавить vlan2 в зону файрвола lan
# Сначала проверь что @zone[0] — это lan:
uci get firewall.@zone[0].name
# Если вывело "lan" — добавляй:
uci add_list firewall.@zone[0].network='vlan2'

uci commit network
uci commit dhcp
uci commit firewall
/etc/init.d/network reload
/etc/init.d/firewall reload
```

> Если `uci get firewall.@zone[0].name` вывел не `lan`, посмотри `uci show firewall | grep "name='lan'"` и подставь нужный индекс зоны.

Wi-Fi точку доступа (шаг 2) всё равно проще создать через LuCI — имена радиомодулей у всех разные.

## Решение проблем

| Симптом | Причина | Решение |
|---------|---------|---------|
| Телефон видит сеть, но «Не удалось подключиться» / «Ошибка получения IP» | Интерфейс не добавлен в зону файрвола — DHCP-запросы режутся | Шаг 1.5 — назначить зону `lan` |
| Подключается, IP есть, но интернета нет | Зона без перенаправления в `wan` (если делал отдельную зону) | Добавить разрешение перенаправления `vlan2 → wan` в Сеть → Межсетевой экран |
| Интернет есть, но трафик идёт напрямую, мимо секции | Реальное имя устройства (`phy1-ap1`) не добавлено в `source_network_interfaces` | Шаг 4 — `uci add_list podkop.settings.source_network_interfaces=...` |
| `ifstatus vlan2` ничего не возвращает | Интерфейс не поднялся — Wi-Fi сеть не создана или не применена | Проверить шаг 2, перезагрузить Wi-Fi (Сеть → Беспроводные сети → Перезапустить радио) |
| Поле «Интерфейс секции» не появилось в LuCI | Кэш браузера или патч не применился | Ctrl+Shift+R; проверить `/www/luci-static/resources/view/podkop/section.js` |

## Настройка Wi-Fi интерфейсов (важно!)

По умолчанию Podkop не перехватывает трафик с отдельных Wi-Fi сетей (vlan2, vlan3). Чтобы они работали, нужно добавить **реальные имена устройств** в настройки Podkop.

### Почему два разных имени

В OpenWrt существуют:
- **UCI-имя** — `vlan2`, `vlan3` (используется в конфигах, и в поле «Интерфейс секции»)
- **Имя устройства ядра** — `phy1-ap1`, `phy1-ap2` (реальные сетевые интерфейсы в Linux)

Podkop использует имена устройств для nftables-правил перехвата трафика, поэтому нужно добавить именно их в `source_network_interfaces`.

### Узнать реальное имя интерфейса

```sh
# Узнать реальное имя для UCI vlan2:
ifstatus vlan2 | jsonfilter -e '@.l3_device'
# → например: phy1-ap1

ifstatus vlan3 | jsonfilter -e '@.l3_device'
# → например: phy1-ap2
```

Или посмотреть все интерфейсы:
```sh
ip link show | grep -E '^[0-9]+:' | awk '{print $2}' | tr -d ':'
```

### Добавить интерфейсы в Podkop

```sh
# Замени phy1-ap1 / phy1-ap2 на имена, которые вернул ifstatus у тебя
uci add_list podkop.settings.source_network_interfaces=phy1-ap1
uci add_list podkop.settings.source_network_interfaces=phy1-ap2
uci commit podkop
/etc/init.d/podkop restart
```

После этого устройства в этих Wi-Fi сетях получат доступ в интернет через Podkop, и поле **«Интерфейс секции»** заработает корректно.

## Совместимость

Установщик ищет точки вставки **по якорям в коде**, а не по номеру версии — поэтому патч переживает минорные обновления Podkop:

- Если найдены оба якоря — работают оба режима
- Если якорь режима «Только списки» не найден (другая версия Podkop) — установщик предупредит и поставит только режим «Весь трафик подсети»
- Если не найден основной якорь — установка прервётся с понятной ошибкой, оригинальные файлы не тронуты

| Версия Podkop | Статус |
|--------------|--------|
| v0.7.17 | ✅ Протестировано |
| v0.7.19 | ✅ Протестировано (оба режима) |
| v0.7.x другие | ⚠️ Должно работать |

> Если Podkop обновился и перестало работать — запусти `uninstall.sh`, обнови Podkop, затем снова `install.sh`.
